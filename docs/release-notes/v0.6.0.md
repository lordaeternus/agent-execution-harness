# v0.6.0

Adds weak model execution support for lower-reasoning agents without increasing token cost.

- Plan compiler rejects vague or oversized tasks before execution.
- `--mode weak` narrows execution to compact next actions, <=2 files/task and typed evidence.
- Failed `verify` returns a short repair hint with a 3-attempt stop condition.
- Evidence policy can require explicit `file_scope`.
- Token benchmark reports standard vs weak next-output size.

Rollback: downgrade to v0.5.0 and remove optional `weak_model` config fields.
