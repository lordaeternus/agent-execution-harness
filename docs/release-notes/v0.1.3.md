# Agent Execution Harness v0.1.3

`v0.1.3` adds stricter evidence enforcement so agents cannot turn partially validated work into a clean `completed` result.

## Highlights

- evidence policy engine for task-level required proof
- `partial_validated` status when required evidence is missing
- evidence quality score in final reports
- `evidence_type` and `evidence_types` fields for structured proof
- UI/layout tasks require focused tests, scoped lint, scoped typecheck and browser smoke or visual assertion
- release and CI workflows updated for Node 24-era action versions

## Why It Matters

The agent should not decide whether it proved the work. The harness should.

With this release, a UI/layout task that skips browser smoke or visual assertion can still produce a report, but it is marked `partial_validated` instead of `completed`.

## Install

```bash
npx agent-execution-harness@latest init --adapter generic --cwd .
```

## Verify

```bash
npx agent-execution-harness@latest doctor --cwd .
```
