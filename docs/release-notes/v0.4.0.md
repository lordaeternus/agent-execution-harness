# v0.4.0 Release Notes

## Codebase Memory

v0.4.0 adds a selective codebase memory layer.

The goal is simple: agents should understand the affected surface before coding, without rereading the whole repository every time.

New commands:

```bash
agent-harness map init
agent-harness map status
agent-harness map query --surface auth
agent-harness map update --files src/auth/session.ts
agent-harness map record --surface auth --files src/auth/session.ts --summary "Auth session owns login state contracts and must be checked before authorization edits."
```

The memory is a cache, not source of truth. Code still wins over docs, docs win over cache, and cache wins over chat history.

## Freshness

The harness stores hashes per file. When a tracked file changes, the related surface becomes `stale` until the agent records a durable summary.

## Subagent Safety

Read-only subagent discoveries can be recorded only with source files, confidence, and main-agent validation.

## Token Impact

Agents can query compact surface summaries instead of loading long docs or broad file sets for every task.
